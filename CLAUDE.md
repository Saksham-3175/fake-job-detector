# CLAUDE.md — Fake Job Detector

## Project Identity
- **Repo**: fake-job-detector
- **Purpose**: ML model that classifies job listings as real/fake. Feeds into the FreshR platform's trust scoring pipeline.
- **Course**: MLOps (academic submission, presentation April 6 2026)
- **Owner**: Saksham (solo dev, 3rd year AI/ML student)

## What This Project Is
A binary classifier trained on the EMSCAD dataset (17,880 listings, 4.8% fake) using TF-IDF + Logistic Regression. The model is served via FastAPI, scraped jobs are classified in real-time, and results are displayed in a Streamlit dashboard. The MLOps lifecycle wraps the entire pipeline: data versioning (DVC), experiment tracking (MLflow), CI/CD with model quality gates (GitHub Actions), and containerized serving (Docker).

## Key Design Decisions
1. **TF-IDF + LR over DistilBERT**: Trains in seconds, no GPU, interpretable, single joblib file. Right tool for this dataset size.
2. **Recall on fake class as north star**: Missing a scam = real harm. False positives (flagging real jobs) = minor inconvenience.
3. **class_weight="balanced"**: Handles 95/5 class imbalance without SMOTE overhead.
4. **Rule-based trust scoring + ML signal**: ML model is one input to the score, not the whole score. Explainability matters.
5. **MLflow over DataDog**: Nothing to monitor pre-launch. MLflow covers experiment tracking + model registry.

## MLOps Lifecycle (The Story for the Prof)
1. **Data Versioning (DVC)**: Raw dataset + splits + model artifact tracked with DVC. Google Drive as remote storage.
2. **Experiment Tracking (MLflow)**: Every training run logs params (C, max_features, ngram_range, class_weight), metrics (accuracy, precision/recall/F1 per class), and artifacts (model joblib, confusion matrix).
3. **Model Registry (MLflow)**: Best model registered, staged (Staging → Production).
4. **CI/CD (GitHub Actions)**: On PR → install → preprocess → train → assert F1_fake > 0.70. Blocks merge if model degrades.
5. **Containerized Serving (Docker)**: FastAPI + Streamlit in docker-compose. Model loaded at startup.
6. **Monitoring (Designed)**: Inference time logged per prediction. Prometheus + Grafana planned for prediction drift detection.
